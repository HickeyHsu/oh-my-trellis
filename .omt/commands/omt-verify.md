# /omt-verify

Verify the active OMT task against requirements and specs.

## Steps

1. Read `execute.md`, task requirements, and specs
2. Run relevant checks or tests
3. Write or update `verify.md` using append-only `## Round N` sections
4. Produce one canonical outcome: `pass`, `fail`, or `human-needed`
5. If outcome is `fail`, execution must loop before close
